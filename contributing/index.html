Contributing

Maxine contributions are expected to update code, tests, and documentation together.

Start with the repository contribution guide:

  • Root CONTRIBUTING.md

The most important rule is simple:

  • if behavior changes, update README.md on GitHub, the relevant docs pages, and api-specs/swagger.yaml on GitHub when the HTTP contract changes

Highest-Value Contribution Areas

  • distributed control-plane hardening beyond Redis lease leadership
  • deeper proxy behavior, retries, and policy controls
  • stronger upstream health semantics
  • external identity, stronger secret handling, and audit improvements
  • dashboards, tracing integrations, and alert-routing guidance
  • frontend testing and UI hardening
  • SDK ergonomics, versioning, and release automation

For the detailed backlog and file-level starting points, use the root contribution guide.
